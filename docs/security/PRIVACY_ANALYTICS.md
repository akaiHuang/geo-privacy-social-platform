# 隱私政策與數據追蹤說明

## 📊 數據收集目的

BroBro 使用 Firebase Analytics 收集匿名化的用戶行為數據，用途包括：

### 1. 產品優化
- 了解用戶最常使用的功能
- 發現用戶體驗中的痛點
- 優化應用性能和穩定性

### 2. 內容推薦
- 分析用戶興趣偏好
- 提供個性化內容推薦
- 改善動態牆排序算法

### 3. 未來商業化準備
- 為潛在廣告服務收集受眾分析數據
- 建立用戶畫像（匿名化）
- 評估功能的商業價值

## 🔒 隱私保護措施

### 我們收集的數據
- ✅ 頁面瀏覽記錄（哪些功能被使用）
- ✅ 互動行為（點讚、收藏、評論次數）
- ✅ 會話時長和活躍度
- ✅ 錯誤日誌（用於修復 bug）
- ✅ 內容創建統計（發文頻率、類型）

### 我們不會收集的數據
- ❌ 貼文具體內容（文字、圖片）
- ❌ 私密對話內容
- ❌ 精確的地理位置（僅統計區域）
- ❌ 第三方平台賬號資訊
- ❌ 設備通訊錄或相冊

### 數據匿名化
- 所有數據通過 Firebase Analytics 匿名化處理
- 無法追溯到特定個人身份
- 符合 GDPR 和相關隱私法規

## 🎯 追蹤的事件類型

### 頁面瀏覽
```typescript
// 範例：用戶進入地圖頁面
{
  page_name: "地圖A",
  view_type: "map",
  timestamp: "2025-10-26T10:30:00Z"
}
```

### 貼文互動
```typescript
// 範例：用戶按讚某篇貼文
{
  action: "like",
  post_id: "abc123",
  post_owner_id: "user456",
  timestamp: "2025-10-26T10:31:00Z"
}
```

### 內容創建
```typescript
// 範例：用戶發布新貼文
{
  content_type: "post",
  has_image: true,
  content_length: 128,
  has_location: true,
  timestamp: "2025-10-26T10:32:00Z"
}
```

### 社交互動
```typescript
// 範例：用戶發送好友邀請
{
  action: "send_friend_request",
  target_user_id: "user789",
  timestamp: "2025-10-26T10:33:00Z"
}
```

### 地圖互動
```typescript
// 範例：用戶使用時光旅行功能
{
  action: "time_travel",
  time_slot: "morning",
  timestamp: "2025-10-26T10:34:00Z"
}
```

## 📈 數據使用場景

### 場景 1：優化按鈕位置
**問題**：「發文」按鈕點擊率低  
**數據**：分析頁面瀏覽時長 vs 互動次數  
**行動**：調整按鈕位置和顏色

### 場景 2：改善地圖性能
**問題**：地圖頁面載入慢  
**數據**：追蹤地圖互動事件和錯誤日誌  
**行動**：優化地圖渲染邏輯

### 場景 3：內容推薦算法
**問題**：用戶停留時間短  
**數據**：分析用戶按讚、收藏、評論的內容類型  
**行動**：推薦類似內容到動態牆頂部

### 場景 4：廣告投放（未來）
**問題**：如何精準投放廣告而不騷擾用戶  
**數據**：分析用戶活躍時段、興趣標籤、互動偏好  
**行動**：在合適的時間、位置展示相關廣告

## 🛡️ 用戶權利

### 您可以
1. **查看數據**：通過 Firebase Console 查看自己的數據統計
2. **退出追蹤**：在設置中關閉數據收集（未來功能）
3. **刪除數據**：聯繫我們刪除歷史數據
4. **導出數據**：請求導出您的所有數據

### 數據保留期限
- 活躍用戶：無限期保留（匿名化）
- 已刪除賬號：30 天後完全清除
- 匯總統計數據：長期保留（無法追溯個人）
